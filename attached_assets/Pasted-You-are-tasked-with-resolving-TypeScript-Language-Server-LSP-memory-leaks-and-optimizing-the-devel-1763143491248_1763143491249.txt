You are tasked with resolving TypeScript Language Server (LSP) memory leaks and optimizing the development environment for a large TypeScript project (8,000+ modules). Follow these explicit rules and do NOT introduce unnecessary code, files, or dependencies. Make only configuration-level changes. System stability and minimalism are critical.

# Instructions

## Immediate Actions
1. **Optimize Configuration (5-10 minutes)**
   - Set `skipLibCheck: true` and `incremental: true` in `tsconfig.json` under `"compilerOptions"`.
   - Set `maxNodeModuleJsDepth: 0` and `skipDefaultLibCheck: true` if not present.
   - Add an `"exclude"` array in `tsconfig.json` with at least:
     ```
     [
       "node_modules",
       "**/dist",
       "**/build",
       "**/__generated__",
       "**/documentation",
       "coverage"
     ]
     ```
   - Add a `"watchOptions"` object:
     ```
     {
       "excludeDirectories": [
         "**/node_modules",
         "**/dist",
         "**/build",
         "**/.cache",
         "**/__generated__",
         "**/documentation"
       ],
       "excludeFiles": [
         "**/*.spec.ts",
         "**/*.test.ts",
         "**/*.log",
         "**/*.map"
       ],
       "watchFile": "useFsEventsOnParentDirectory",
       "watchDirectory": "useFsEventsOnParentDirectory"
     }
     ```

## Short-term Actions (15-30 minutes)
2. **Validate File Inclusion**
   - Run `tsc --listFilesOnly` and ensure only legitimate `src/**` (source) files are matched.
   - Confirm no build artifacts, documentation, or cache directories are processed or watched.
   - If `.d.ts` or build files appear, adjust the `"exclude"` in `tsconfig.json` accordingly.

3. **LSP Memory Monitoring**
   - Document in `replit.md` and project docs: Developers should reload the IDE window (Ctrl+R/Cmd+R) if slowdowns occur.
   - Optionally, set a scheduled reminder in the team's workflow to restart the IDE every 4-6 hours during long sessions.

## Long-term/Optional (only if memory still grows):
4. **TypeScript Project References**
   - Only if configuration changes above do NOT reduce memory: Refactor into smaller projects using TypeScript Project References (`tsconfig.json` with `"references"` key).
   - Each package/module receives its own `tsconfig.json` and is independently type-checked.

## Final Validation
5. **Monitoring**
   - Track the `typescript-language-server` memory usage. Expected: Drop from ~2.3GB each to <800MB.
   - IDE autocomplete should return in <1 second. No frequent full re-indexing on every file save.

- Do NOT add new npm dependencies.
