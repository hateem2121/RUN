# Accessibility (A11y) Testing

We use `@axe-core/playwright` to automatically catch accessibility violations in CI.

## Local Running

1. **Ensure dev server is running:**
   ```bash
   npm run dev
   ```
2. **Run a11y tests:**
   ```bash
   npx playwright test tests/e2e/a11y.spec.ts
   ```

## CI Integration

A11y tests are run as part of the `e2e` job in GitHub Actions. Failure to meet WCAG 2.1 AA standards will fail the build.

## Common Fixes

- **Missing `aria-label`**: Ensure interactive elements have descriptive labels.
- **Color Contrast**: Use OKLCH color space for accessible contrast ratios.
- **Focus States**: Don't remove default focus outlines without providing a visible alternative.
