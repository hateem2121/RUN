### C2 · Remove try/catch blocks from Express 5 route handlers

**Constitutional rule:** Express 5 catches async errors automatically — `try/catch` in route handlers is forbidden.

**Analysis of actual route-level try/catch blocks (from exploration):**

| File | Line | Type | Action |
|------|------|------|--------|
| `server/routes/utilities/metrics.ts` | 443 | Route handler: `/api/health/db` wraps entire body | **REMOVE** — convert to Express 5 pattern |
| `server/routes/resources/sustainability.routes.ts` | 100 | Inside `.map()` callback for schema validation tests | **KEEP** — this is Zod schema testing logic, not route-level |
| `server/routes/utilities/api-based-population.ts` | 72,128,225,325,457 | Inside `for...of` loops for batch population | **KEEP** — these are per-item error handling in a batch operation, designed to continue on failure |
| `server/routes/media/utils.ts` | 307,584,596,628,648 | Utility functions (MediaUrlResolver, processUploadedFile) | **KEEP** — internal operation try/catch, not route handlers |
| `server/routes/media/services.ts` | 330,397,486,512,533 | Service methods (GLTF validation, DB creation with cleanup) | **KEEP** — compensating transaction patterns |
| `server/routes/media/handlers.ts` | 391,414,471,484,523,549,826,835,944,1342 | Internal operations (image processing, cleanup, JSON parsing) | **KEEP** — all are specific operation handlers, not route-level wrappers |

**Decision confirmed with user: Only remove true route-level wrappers. Keep all 26 internal operation try/catch blocks.**

**Only ONE file needs actual route-level try/catch removal:**

#### File: `server/routes/utilities/metrics.ts` — `/api/health/db` endpoint (lines 442-466)

**Before:**

### H2 · Split Kubernetes health probes into liveness and readiness

**File:** `k8s/argocd/base/deployment.yaml`

Currently both probes hit `/api/health`. Split them:

```yaml
livenessProbe:
  httpGet:
    path: /healthz
    port: 5002
  initialDelaySeconds: 15
  periodSeconds: 20
  failureThreshold: 3

readinessProbe:
  httpGet:
    path: /readyz
    port: 5002
  initialDelaySeconds: 10
  periodSeconds: 10
  failureThreshold: 3
```

**File:** `server/boot/middleware.ts` — add routes in `setupHealthChecks()`:

```typescript
export function setupHealthChecks(app: Express) {
  // Existing simple health check (backward compatible)
  app.get("/api/health", (_req, res) => {
    res.json({ status: "UP", timestamp: new Date().toISOString(), uptime: process.uptime() });
  });

  // Liveness probe — lightweight, no dependencies
  app.get("/healthz", (_req, res) => {
    res.json({ status: "alive", ts: Date.now() });
  });

  // Readiness probe — full dependency check
  app.get("/readyz", async (_req, res, next) => {
    const { checkDatabaseConnection } = await import("../db.js");
    const dbOk = await checkDatabaseConnection();

    let cacheOk = true;
    try {
      const { UnifiedCache } = await import("../lib/cache/unified-cache.js");
      const health = await UnifiedCache.getHealthStatus();
      cacheOk = health.healthy;
    } catch {
      cacheOk = false;
    }

    if (!dbOk || !cacheOk) {
      res.status(503).json({ status: "not_ready", db: dbOk, cache: cacheOk });
      return;
    }
    res.json({ status: "ready", db: dbOk, cache: cacheOk });
  });
}
```

### H6 · Fix logout route — proper session destruction

**File:** `server/routes/auth.ts` (lines 65-72)

**Before:**
```typescript
router.get("/logout", (req, res, next) => {
  req.logout((err) => {
    if (err) {
      return next(err);
    }
    res.redirect("/");
  });
});
```

**After:**
```typescript
router.post("/logout", (req, res, next) => {
  req.logout((err) => {
    if (err) {
      return next(err);
    }
    req.session.destroy((destroyErr) => {
      if (destroyErr) {
        return next(destroyErr);
      }
      res.clearCookie("connect.sid", {
        path: "/",
        httpOnly: true,
        secure: process.env.NODE_ENV === "production",
        sameSite: "lax",
      });
      res.redirect("/");
    });
  });
});
```

Note: Changed from GET to POST (OWASP best practice — logout should be a mutating operation).

**Verify:**
```bash
npm run typecheck --workspace=@run-remix/server
```
